# Тестовое задание: Развертывание веб-приложения через Nginx в Docker

Проект представляет собой простое веб-приложение на Python, работающее за реверс-прокси Nginx внутри изолированной сети Docker.

## Структура проекта
├── backend/
│   ├── Dockerfile
│   └── app.py
├── nginx/
│   └── nginx.conf
├── docker-compose.yml
└── README.md

## Как запустить проект

1. Убедитесь, что у вас установлены `docker` и `docker-compose`.
2. Клонируйте репозиторий и перейдите в корень проекта.
3. Запустите сборку и старт контейнеров в фоновом режиме:
   ```bash
   docker-compose up -d --build
    ```

## Как работает схема 
1. Изоляция: Контейнеры запускаются в кастомной bridge-сети Docker (app-network).

2. Входная точка: Порт 80 контейнера nginx проброшен на хост-машину. Запросы извне (например, от curl) попадают на Nginx.

3. Защита бэкенда: Порт 8080 контейнера backend наружу не публикуется - к нему невозможно обратиться напрямую с хоста, что обеспечивает безопасность.

4. Проксирование: Nginx принимает запрос на порту 80, парсит путь / и, благодаря встроенному DNS-резолверу Docker, перенаправляет запрос на upstream backend, используя имя сервиса backend:8080. Python-сервер обрабатывает запрос и возвращает текст обратно через Nginx пользователю.
